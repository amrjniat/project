## Plan: موقع HTML5 متعدد الصفحات

تنفيذ موقع تعليمي من 5 صفحات مستقلّة بواجهة عربية متجاوبة، مع هيكلة HTML5 واضحة، تصميم CSS3 مخصص فوق Bootstrap 5، وتفاعلات JavaScript بسيطة ومختلفة في كل صفحة. بما أن مساحة العمل فارغة، الأفضل البدء ببنية ملفات مشتركة ثم بناء الصفحات فوقها لتجنب التكرار وضمان اتساق التصميم والسلوك.

**Steps**
1. إنشاء هيكل المشروع الأساسي: `index.html` كصفحة روابط رئيسية، ومجلد `pages/` لصفحات المحتوى، ومجلدات `assets/css/` و`assets/js/` و`assets/img/` و`assets/icons/` عند الحاجة.
2. إعداد القالب المشترك لكل الصفحات باستخدام عناصر HTML5 المطلوبة: `header`, `main`, `section`, `footer`، مع `title` مناسب و`meta viewport` و`meta description`.
3. ربط Bootstrap 5 عبر CDN في البداية لتسريع التنفيذ، ثم إضافة ملف CSS خارجي مخصص لتجاوز Bootstrap عند الحاجة وضبط الألوان والخطوط والمسافات والهوية البصرية.
4. بناء شريط تنقل موحّد `Navbar` في جميع الصفحات مع روابط واضحة إلى الصفحات الخمس، والتأكد أن `index.html` يجمع كل الروابط المطلوبة بشكل مباشر.
5. تنفيذ الصفحات الخمس واحدة تلو الأخرى مع محتوى مناسب للمبتدئين وتوزيع Bootstrap Grid وCards وButtons وForms حسب الحاجة:
   - صفحة Portfolio: صورة/بطاقات مشاريع/أزرار تواصل/زر تبديل داكن-فاتح.
   - صفحة Blog: 3 أو 4 مقالات قصيرة/تواريخ نشر/أزرار قراءة المزيد.
   - صفحة Product Gallery: منتجات داخل Cards مع عداد للسلة.
   - صفحة Calculator: حاسبة تعمل بالكامل بـ JavaScript.
   - صفحة Consultation Form: حقول اسم وبريد ورسالة مع تحقق JavaScript ومنع الإرسال عند الفراغ.
6. كتابة JavaScript بحيث يعمل تفاعل واحد على الأقل في كل صفحة بدون أخطاء في الكونسول.
7. تحسين التجاوب لكل المقاسات باستخدام Bootstrap grid مع تعديلات CSS إضافية للجوال والتابلت والحاسوب، ومراجعة التباعد وحجم الخطوط والأزرار والصور.
8. إجراء فحص نهائي لكل صفحة للتأكد من عدم وجود أخطاء في الكونسول، وأن كل تفاعل يعمل، وأن الروابط من `index.html` صحيحة، وأن الواجهة تبدو متسقة على الشاشات المختلفة.

**Relevant files**
- `index.html` — صفحة الدخول وروابط الصفحات الخمس.
- `pages/portfolio.html` — صفحة السيرة الذاتية التفاعلية.
- `pages/blog.html` — صفحة المدونة البسيطة.
- `pages/products.html` — صفحة معرض المنتجات الوهمي.
- `pages/calculator.html` — صفحة الحاسبة.
- `pages/consultation.html` — صفحة نموذج الاستشارة.
- `assets/css/style.css` — التعديلات الخاصة فوق Bootstrap والهوية البصرية والتجاوب.
- `assets/js/main.js` — التفاعلات العامة ومنطق الصفحات.

**Verification**
1. فتح كل صفحة والتأكد من ظهور `header`, `main`, `section`, `footer` بشكل صحيح وعدم كسر التخطيط.
2. اختبار التفاعلات الخمس: تبديل السمة، قراءة المزيد، إضافة للسلة، الحاسبة، والتحقق من النموذج.
3. فحص التجاوب على أحجام شاشة الجوال والتابلت والحاسوب باستخدام أدوات المعاينة.
4. التأكد من أن `index.html` يحتوي على روابط تعمل لكل الصفحات وأن التنقل لا يعطي أخطاء.
5. مراجعة الكونسول للتأكد من عدم وجود أخطاء JavaScript.

**Decisions**
- اعتماد مشروع متعدد الصفحات وليس صفحة واحدة، لأن المتطلبات تذكر صفحة `index.html` وروابط لصفحات مستقلة.
- استخدام Bootstrap 5 عبر CDN كخيار افتراضي أسرع وأسهل للمبتدئ، مع ملف CSS خارجي لتخصيص الشكل.
- جعل JavaScript مشتركًا لكن مقسّم السلوك حسب الصفحة، بدل نسخ كود منفصل بالكامل لكل صفحة.
- إبقاء المحتوى عربيًا متسقًا لتناسب طلب المستخدم والشرح التعليمي.
